
# Data Catalog: Gold Layer Schema

## 1. Overview
This schema follows a **Star Schema** design, where `fact_sales` serves as the central transaction table, supported by `dim_customers` and `dim_products` for descriptive analysis.

### Relationship Diagram
* **`gold.fact_sales`** (customer_key) ↔ **`gold.dim_customers`** (customer_key)
* **`gold.fact_sales`** (product_key) ↔ **`gold.dim_products`** (prodcut_key)

---

## 2. Fact Tables

### gold.fact_sales
*Purpose: Tracks transactional sales activity, quantities, and timing.*

| Column | Type | Description |
| :--- | :--- | :--- |
| `order_key` | INT | Primary Key: Unique identifier for the order line item. |
| `order_number` | NVARCHAR(50) | The business reference number for the order. |
| `customer_key` | INT | FK: Links to `dim_customers`. |
| `product_key` | INT | FK: Links to `dim_products`. |
| `order_date` | DATE | Date the order was submitted. |
| `ship_date` | DATE | Date the order was dispatched. |
| `duration_date` | DATE | Fulfillment/completion milestone date. |
| `sales` | INT | Total revenue generated by the line item. |
| `quantity` | INT | Number of units purchased. |
| `price` | INT | Unit price at time of purchase. |

---

## 3. Dimension Tables

### gold.dim_customers
*Purpose: Stores demographic and identifying information about customers.*

| Column | Type | Description |
| :--- | :--- | :--- |
| `customer_key` | INT | Primary Key: Unique identifier for the customer. |
| `customer_id` | INT | The source system identifier. |
| `customer_code` | NVARCHAR(50) | Business-specific customer code. |
| `first_name` | NVARCHAR(50) | Customer first name. |
| `last_name` | NVARCHAR(50) | Customer last name. |
| `gender` | NVARCHAR(50) | Demographic gender (E.g:Male, Female, N/A). |
| `martial_status` | NVARCHAR(50) | Marital status for analytics. |
| `country` | NVARCHAR(50) | Geographic location (Country). |
| `brith_day` | NVARCHAR(50) | Customer date of birth. |

### gold.dim_products
*Purpose: Stores product definitions, category hierarchies, and cost metrics.*

| Column | Type | Description |
| :--- | :--- | :--- |
| `prodcut_key` | INT | Primary Key: Unique identifier for the product. |
| `product_id` | INT | Source system identifier. |
| `product_number` | NVARCHAR(50) | Structured alphnumeric code represent the product. |
| `product_name` | NVARCHAR(50) | Descriptive name of the product. |
| `category_id` | NVARCHAR(50) | Unique identifier for the product. |
| `category_name` | NVARCHAR(50) | The main business category. |
| `subcategory` | NVARCHAR(50) | The sub-level of the product hierarchy. |
| `product_line` | NVARCHAR(50) | Business product line division. |
| `product_cost` | INT | Unit cost for profit calculation. |
| `maintenance` | NVARCHAR(50) | Maintenance status, if maintenance is allowed or not (e.g Yes, No). |
| `start_date` | DATE | Date the product was added to the catalog. |
